# Check Navigation Button Names

指定された画面の遷移ボタンが、遷移先の画面内容に適した名称になっているか確認し、必要に応じて修正を提案するスキルです。

## 使い方

```
/check-navigation-button <画面ファイルパスまたは画面名>
```

## 実行手順

1. **現在の画面を特定**
   - 指定されたファイルまたは画面名から対象の画面ファイルを読み取る
   - 画面内の遷移ボタン（FilledButton, OutlinedButton, ElevatedButton等）を特定する

2. **遷移先を特定**
   - ボタンの `onPressed` で呼ばれる処理を確認（`advancePhase`, `Navigator.push`, `context.go` 等）
   - ゲームフローの場合は `game_screen.dart` のフェーズ定義を確認
   - ルーティングの場合は `router.dart` を確認

3. **遷移先の画面内容を確認**
   - 遷移先の画面ファイルを読み取る
   - AppBarのtitle、画面の主要コンテンツ、目的を把握する

4. **ボタン名称を評価**
   - 現在のボタンテキスト（ローカライゼーションキー）を確認
   - 遷移先の画面内容と照らし合わせて適切かどうか判断
   - 不適切な場合は、遷移先の内容に基づいた名称を提案

5. **修正の実施**
   - ユーザーの承認を得て、ローカライゼーションファイルの更新
   - 画面ファイルのボタンテキスト参照を更新

## チェックポイント

- [ ] ボタン名が遷移先の画面タイトルや目的と一致しているか
- [ ] ユーザーが次に何をするか明確に伝わるか
- [ ] 日本語と英語の両方のローカライゼーションが適切か
- [ ] 既存のローカライゼーションキーで適切なものがないか確認

## 出力形式

```
## 確認結果

### 現在の画面
- ファイル: `path/to/current_screen.dart`
- 遷移ボタン: `context.tr('current_key')` → "現在のテキスト"

### 遷移先の画面
- ファイル: `path/to/next_screen.dart`
- 画面タイトル: "○○○"
- 画面の目的: ○○○を行う

### 評価
- 適切 / 要修正

### 提案（要修正の場合）
- 推奨キー: `new_key` または既存キー `existing_key`
- 日本語: "○○へ進む"
- 英語: "Proceed to ○○"
```
